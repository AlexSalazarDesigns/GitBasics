What is Version Control?



A practice or method of recording changes made to a project that distinguishes between distinct updates and allows users to review, modify or rollback to prior versions of code.

Version control is not the same thing as backing up a file. In version control you are specifically tracking changes made throughout the development of a project. Backing up stores a copy of the fully functioning file in case it is lost.

B.G. - Before Git

Once upon a time. . .


Image Credit: Smashing Magazine
Centralized system - the main (production) copy of the program exists in a centralized location. If you want to work on it, you ‘check out’ a copy and a message is posted that automatically tells the next person who has it checked out first. Programmer A and B have to coordinate their changes. Either programmer A loads her changes and then programmer B checks it out and goes again, or in the interest of time, A/B may work together to merge their changes and load up the program at once. This is both time consuming and inefficient, and muddies the pedigree of changes. What if A’s changes work but something unrelated about B’s causes an issue? Can you back out A’s changes without backing out B’s?

The Rise of Git


Image Credit: Smashing Magazine

Early versions included Mercurial and Bazaar, but Git, developed by Linus Torvold (also the inventor and vision behind the Linux operating system on which Android technology is based) has quickly overtaken all of them.

Git vs. Git Hub



Git is the version control concept and system.


Git Hub is one of several tools developers may use to manage a project using Git.

Exploring the Interfaces

Git Bash does not universally recognize commands from other frameworks. You can execute git commands from your command line prompt, but you cannot always execute other commands (like Meteor) from GitBash. Functionally, however, one command prompt is much like another.
